<task type="auto">
  <name>Task 1: Install rss-parser and build RSS API route</name>
  <files>app/api/widgets/rss/route.ts, package.json</files>
  <action>
Install the rss-parser package: `npm install rss-parser`

Then create `app/api/widgets/rss/route.ts` with a GET handler.

Query parameters:
- `url` (required): URL-encoded RSS/Atom feed URL
- `limit` (optional): max number of items to return, default 20

Handler logic:
1. Extract and validate the `url` parameter. Return 400 if missing.
2. Validate the URL is a valid URL (use `new URL()` in a try/catch). Return 400 if invalid.
3. Import and use rss-parser:
   ```typescript
   import Parser from "rss-parser"

   const parser = new Parser({
     timeout: 10000,
     headers: {
       "User-Agent": "Homelabarr/1.0",
       Accept: "application/rss+xml, application/atom+xml, application/xml, text/xml",
     },
   })
   ```
4. Parse the feed: `const feed = await parser.parseURL(url)`
5. If parsing fails, return 502 with `{ error: "Failed to fetch or parse feed" }`
6. Transform the response to a clean shape:
   ```typescript
   {
     title: feed.title,
     description: feed.description,
     link: feed.link,
     items: feed.items.slice(0, limit).map(item => ({
       title: item.title ?? "Untitled",
       link: item.link ?? "",
       pubDate: item.pubDate ?? item.isoDate ?? "",
       contentSnippet: (item.contentSnippet ?? "").slice(0, 200),
       creator: item.creator ?? item["dc:creator"] ?? "",
     }))
   }
   ```
7. Return the transformed feed as JSON with `Cache-Control: public, max-age=300` header (5-minute cache).

IMPORTANT: The rss-parser package may not have built-in TypeScript types. If `import Parser from "rss-parser"` causes type issues, use `const Parser = require("rss-parser")` or add a type declaration. Check if `@types/rss-parser` exists — if not, the package includes its own types.

Actually, rss-parser DOES include its own TypeScript types. Use the standard import.

  <done>rss-parser installed. GET /api/widgets/rss fetches and parses RSS/Atom feeds server-side, returns normalized feed items with 5-minute cache. Returns 400 for missing/invalid URL, 502 for fetch/parse failures.</done>

<task type="auto">
  <name>Task 2: Build RssWidget component</name>
  <files>components/widgets/rss-widget.tsx</files>
  <action>
Create `components/widgets/rss-widget.tsx` as a `"use client"` component.

Props interface:
```typescript
interface RssWidgetProps {
  widgetId: string
  config: Record<string, unknown> | null
}
```

Config shape (from props.config):
- `feeds`: array of feed objects `{ url: string, name?: string }` (default: `[]`)
- `maxItems`: number (max items to show total across all feeds, default: 15)
- `refreshInterval`: number (refresh interval in minutes, default: 15)

Component states:
1. **Setup mode** (no feeds configured): Show a form to add the first feed URL with a name.
2. **Loading state**: Show skeleton placeholders while fetching feeds.
3. **Display mode**: Show aggregated feed items from all configured feeds.
4. **Error state**: Show which feeds failed with retry option.

Data fetching:
- Use `useEffect` to fetch all configured feeds in parallel: `Promise.allSettled(feeds.map(f => fetch(`/api/widgets/rss?url=${encodeURIComponent(f.url)}&limit=20`)))`
- Merge all feed items, sort by pubDate (newest first), limit to maxItems
- Refresh on the configured interval (default 15 minutes)
- Track per-feed error status to show which feeds are failing

Display layout:
- Header bar: RSS icon (RssIcon or similar from hugeicons) + "RSS Feed" title + settings gear button
- Feed items list (scrollable):
  - Each item: title (clickable link, opens in new tab), source feed name (small badge), relative time ("2h ago" using date-fns `formatDistanceToNow`), content snippet (truncated, muted text)
  - Items separated by thin borders
  - On hover, highlight the item row
- If a feed has errors, show a small warning indicator next to that feed's items

Settings panel (toggled by gear icon):
- List of configured feeds with URL and optional name, each with a delete (X) button
- "Add Feed" form at the bottom: URL input + optional name input + Add button
- Max items slider/input (5-50 range)
- Save button that PATCHes the full config

Feed item click behavior: `window.open(item.link, "_blank", "noopener,noreferrer")`

Date formatting:
- Use `formatDistanceToNow(new Date(item.pubDate), { addSuffix: true })` from date-fns for relative times (e.g., "2 hours ago")
- If pubDate is invalid/missing, show "Unknown date"

Visual style:
- Outer div: `h-full w-full flex flex-col rounded-lg border border-border bg-card overflow-hidden`
- Header bar: standard pattern with icon + title + gear
- Feed items: `flex-1 overflow-y-auto` container
- Each item: `px-3 py-2 border-b border-border hover:bg-muted/50 cursor-pointer` with title, meta line (source + time), and snippet
- Source badge: small pill with feed name `text-[0.625rem] bg-muted px-1.5 py-0.5 rounded-full`
