Open-Meteo API reference:
- Current weather: `https://api.open-meteo.com/v1/forecast?latitude=X&longitude=Y&current=temperature_2m,relative_humidity_2m,apparent_temperature,weather_code,wind_speed_10m&temperature_unit=celsius`
- Daily forecast: add `&daily=temperature_2m_max,temperature_2m_min,weather_code&timezone=auto`
- No API key needed, generous rate limits
- weather_code maps to WMO weather interpretation codes (0=clear, 1-3=partly cloudy, 45-48=fog, 51-55=drizzle, 61-65=rain, 71-75=snow, 80-82=showers, 95-99=thunderstorm)
</context>

  <name>Task 1: Build weather API proxy route</name>
  <files>app/api/widgets/weather/route.ts</files>
  <action>
Create `app/api/widgets/weather/route.ts` with a GET handler that proxies requests to Open-Meteo.

Query parameters:
- `lat` (required): latitude as number
- `lon` (required): longitude as number
- `unit` (optional): `"celsius"` or `"fahrenheit"`, default `"celsius"`

Handler logic:
1. Extract and validate query parameters. Return 400 if lat/lon are missing or not valid numbers.
2. Build the Open-Meteo URL:
   ```
   https://api.open-meteo.com/v1/forecast?latitude=${lat}&longitude=${lon}&current=temperature_2m,relative_humidity_2m,apparent_temperature,weather_code,wind_speed_10m&daily=temperature_2m_max,temperature_2m_min,weather_code&temperature_unit=${unit}&timezone=auto&forecast_days=5
   ```
3. Fetch from Open-Meteo with a 10-second timeout (use AbortController).
4. If the fetch fails or returns non-OK, return a 502 error with `{ error: "Failed to fetch weather data" }`.
5. Parse the JSON response and return it directly as `NextResponse.json(data)`.
6. Add `Cache-Control: public, max-age=600` header (cache for 10 minutes) to the response.

IMPORTANT: This is a simple passthrough proxy. Do NOT transform the data — let the client component handle the response shape. The Open-Meteo response structure is well-documented and stable.
  </action>

  <done>GET /api/widgets/weather proxies to Open-Meteo with lat/lon/unit parameters, returns weather JSON with 10-minute cache header. Returns 400 for missing params, 502 for upstream failures.</done>

- Wind speed from `wind_speed_10m`

WMO weather code mapping (include this as a helper function):
```typescript
function getWeatherInfo(code: number): { label: string; emoji: string } {
  if (code === 0) return { label: "Clear sky", emoji: "☀️" }
  if (code <= 3) return { label: "Partly cloudy", emoji: "⛅" }
  if (code <= 48) return { label: "Foggy", emoji: "🌫️" }
  if (code <= 55) return { label: "Drizzle", emoji: "🌦️" }
  if (code <= 65) return { label: "Rainy", emoji: "🌧️" }
  if (code <= 75) return { label: "Snowy", emoji: "❄️" }
  if (code <= 82) return { label: "Showers", emoji: "🌧️" }
  if (code <= 99) return { label: "Thunderstorm", emoji: "⛈️" }
  return { label: "Unknown", emoji: "❓" }
}
```

Settings panel (toggled by gear icon):
