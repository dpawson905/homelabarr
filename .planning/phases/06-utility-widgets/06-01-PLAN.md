  <name>Task 1: Build ClockWidget component</name>
  <files>components/widgets/clock-widget.tsx</files>
  <action>
Create `components/widgets/clock-widget.tsx` as a `"use client"` component.

Props interface:
```typescript
interface ClockWidgetProps {
  widgetId: string
  config: Record<string, unknown> | null
}
```

Config shape (read from props.config):
- `timezone`: string (IANA timezone like "America/New_York", default: browser local via `Intl.DateTimeFormat().resolvedOptions().timeZone`)
- `format`: `"12h"` | `"24h"` (default: `"12h"`)
- `showSeconds`: boolean (default: true)
- `showDate`: boolean (default: true)

Component behavior:
1. Use `useState` for current time, initialized to `new Date()`
2. Use `useEffect` with `setInterval(1000)` to update time every second. Clean up on unmount.
3. Format time using `Intl.DateTimeFormat` with the configured timezone and hour12 option (derived from format). This avoids timezone library dependencies.
4. Display the time prominently (large text, centered) and the date below it (smaller text)
5. Include a small settings button (gear icon) in the top-right corner that opens an inline settings panel (useState toggle, not a dialog)
6. The settings panel should have:
   - A text input for timezone (placeholder: "e.g. America/New_York", with a note "Leave empty for local time")
   - A toggle/select for 12h vs 24h format
   - Toggle for show seconds
   - Toggle for show date
   - A Save button that PATCHes the config via `fetch(`/api/widgets/${widgetId}`, { method: "PATCH", body: JSON.stringify({ config: { timezone, format, showSeconds, showDate } }) })`
7. After saving, update local state from the response

Visual style — match existing widget patterns:
- Outer div: `h-full w-full flex flex-col rounded-lg border border-border bg-card`
- Header bar: `flex shrink-0 items-center justify-between border-b border-border px-3 py-2` with icon + "Clock" label
- Time display: large monospace text (use `font-mono` class), centered in the remaining space
- Use HugeiconsIcon with Clock01Icon (or similar) from @hugeicons/core-free-icons for the header icon. Search the icon set for a clock icon — try `Clock01Icon` or `Time01Icon`. If neither exists, use `TimeQuarter01Icon` or any time-related icon available.
- Use Settings02Icon for the settings gear button

IMPORTANT: Use `select` elements with the `@/components/ui/select` shadcn component for the format picker, and simple `@/components/ui/input` for the timezone field. Use `@/components/ui/button` for save. Use checkbox or switch from shadcn for toggles.
  </action>

  <name>Task 2: Build NotesWidget component</name>
  <files>components/widgets/notes-widget.tsx</files>
  <action>
Create `components/widgets/notes-widget.tsx` as a `"use client"` component.

Props interface — same pattern as ClockWidget:
```typescript
interface NotesWidgetProps {
  widgetId: string
  config: Record<string, unknown> | null
}
```

Config shape (read from props.config):
- `content`: string (the notes text, default: `""`)
- `title`: string (widget header title, default: `"Notes"`)

Component behavior:
1. Initialize local state from `config?.content` (cast to string, default `""`)
2. Use a `<textarea>` that fills the widget body area for editing
3. Auto-save: Use a debounced save (500ms after last keystroke) via `useRef` timeout pattern (same pattern used in widget-grid.tsx for layout saves). On each keystroke, clear the previous timeout, set a new 500ms timeout that PATCHes config.
4. Save via: `fetch(`/api/widgets/${widgetId}`, { method: "PATCH", body: JSON.stringify({ config: { content: text, title } }) })`
5. Show a subtle "Saving..." / "Saved" indicator in the header area (use a small text label that appears briefly)
6. The textarea should have no visible border inside the card, just fill the content area. Use `resize-none` and `outline-none` classes. Match the card background.

Visual style:
- Outer div: `h-full w-full flex flex-col rounded-lg border border-border bg-card`
- Header bar with NoteIcon (or similar hugeicon) + title text + save status indicator
- Textarea fills remaining space: `flex-1 w-full p-3 text-sm bg-transparent resize-none outline-none`
- Placeholder text: "Start typing..."

IMPORTANT: Do NOT add a markdown renderer or preview mode. Keep it simple — just a textarea for plain text notes. Markdown rendering can be added in a future phase.
  </action>

  <done>NotesWidget displays an editable textarea that auto-saves content to widgetConfigs via debounced PATCH. Shows save status indicator. Content persists across page reloads.</done>
