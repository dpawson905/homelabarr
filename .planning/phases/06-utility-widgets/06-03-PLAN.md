  <action>
Create `components/widgets/calendar-widget.tsx` as a `"use client"` component.

Props interface:
```typescript
interface CalendarWidgetProps {
  widgetId: string
  config: Record<string, unknown> | null
}
```

Config shape (from props.config):
- `events`: array of event objects (default: `[]`)
- `title`: string (widget header title, default: `"Calendar"`)

Event shape:
```typescript
interface CalendarEvent {
  id: string        // crypto.randomUUID() generated client-side
  title: string
  date: string      // ISO date string "YYYY-MM-DD"
  time?: string     // Optional "HH:mm" format
  description?: string
  color?: string    // Optional accent color (one of a few preset options)
}
```

Component structure — two main views:

**1. Agenda view (default):**
- Shows upcoming events sorted by date/time, grouped by date
- Each date group: date heading (e.g., "Today", "Tomorrow", "Wed, Mar 4") followed by event items
- Each event item: colored left border accent, time (if set), title, description preview
- Only show events from today forward (filter out past events)
- If no upcoming events, show an empty state: "No upcoming events" with an "Add Event" button
- Show at most 10-15 upcoming events to prevent overflow

**2. Add/Edit event form (inline panel, not a separate dialog):**
- Toggle via an "Add" button (PlusSignIcon) in the header
- Fields: title (required), date (required — use a simple date input `<input type="date">`), time (optional — `<input type="time">`), description (optional textarea), color (optional — 5-6 preset color dots to pick from)
- Save button creates/updates the event in the events array and PATCHes the full config
- When editing an existing event, pre-fill the form and show a Delete button
- Cancel button closes the form

Color presets for events (use Tailwind classes):
- Default (teal/primary): `bg-primary`
- Red: `bg-red-500`
- Blue: `bg-blue-500`
- Green: `bg-green-500`
- Yellow: `bg-yellow-500`
- Purple: `bg-purple-500`

Save logic:
- After adding/editing/deleting an event, update the local events state AND PATCH the config:
  ```typescript
  fetch(`/api/widgets/${widgetId}`, {
    method: "PATCH",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ config: { events: updatedEvents, title } }),
  })
  ```

Date formatting — use date-fns:
- `import { format, isToday, isTomorrow, isPast, parseISO, compareAsc } from "date-fns"`
- Group heading: if isToday → "Today", if isTomorrow → "Tomorrow", else `format(date, "EEE, MMM d")`
- Event time display: use the raw time string (already in "HH:mm" format)

Visual style:
- Outer div: `h-full w-full flex flex-col rounded-lg border border-border bg-card overflow-hidden`
- Header bar: calendar icon (Calendar03Icon or similar from hugeicons) + title + add button
- Agenda list: scrollable with `flex-1 overflow-y-auto`
- Event items: left colored border (3px), padding, hover highlight
- Date group headings: sticky, small text, muted color, top border separator

Click on an event to edit it (open the form pre-filled with event data).
