  <action>
Create lib/health-check.ts with a `checkAppHealth` function that:
- Takes a URL string and an optional timeout (default 10 seconds)
- Performs an HTTP HEAD request first (lighter); if HEAD fails or returns non-2xx, falls back to GET
- Uses the native `fetch` with an AbortController for timeout
- Returns an object: `{ status: "online" | "offline" | "degraded", latency: number, statusCode: number | null, checkedAt: string }`
- "online" = 2xx response, "degraded" = 3xx/4xx/5xx response (server reachable but unhealthy), "offline" = network error/timeout
- Latency is measured in milliseconds from request start to response
- Catches all errors gracefully (network errors, DNS failures, timeouts) and returns "offline"

Export the HealthCheckResult type so other files can import it.

Add an in-memory cache (simple Map) in the same file:
- Key: app ID, Value: { result: HealthCheckResult, timestamp: number }
- `getCachedHealth(appId, maxAgeSeconds)` returns cached result if it exists and age < maxAgeSeconds
- `setCachedHealth(appId, result)` stores the result with current timestamp
- This avoids hammering services on every client poll

Create GET /api/apps/health (route.ts):
- Reads all apps from the database using the existing `getApps()` query from lib/db/queries.ts
- Filters to only apps with statusCheckEnabled === 1
- For each enabled app: check cache first (using app's statusCheckInterval), if stale run checkAppHealth, update cache
- Return JSON: `{ statuses: { [appId]: HealthCheckResult } }`
- Run health checks in parallel using Promise.all (fine for homelab scale)
- Also include apps with statusCheckEnabled === 0 as `{ status: "disabled" }` in the response so the client knows their state

Create GET /api/apps/[id]/health (route.ts):
- Read the app by ID from DB using `db.select().from(apps).where(eq(apps.id, id)).get()`
- If app not found, return 404
- If statusCheckEnabled is 0, return `{ status: "disabled" }`
- Otherwise check cache, run check if stale, return the single HealthCheckResult
