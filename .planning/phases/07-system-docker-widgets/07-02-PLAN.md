**Route 1: GET /api/widgets/docker (container list with stats)**

Create `app/api/widgets/docker/route.ts` with a GET handler.

Query parameters:
- `socketPath` (optional): custom Docker socket path (default: `/var/run/docker.sock`)
- `all` (optional): if `"true"`, show all containers including stopped (default: only running)

Handler logic:
1. Create a Dockerode instance:
   ```typescript
   import Docker from "dockerode"
   const docker = new Docker({ socketPath: socketPath || "/var/run/docker.sock" })
   ```
2. First, test the connection with `await docker.ping()`. If this fails, return a structured error:
   ```typescript
   { error: "docker_unavailable", message: "Cannot connect to Docker daemon. Is Docker running?" }
   ```
   with status 503.
3. List containers: `const containers = await docker.listContainers({ all: showAll })`
4. For each running container, get stats snapshot. Use `Promise.allSettled` to avoid one container's stats failure blocking others:
   ```typescript
   const statsPromises = containers
     .filter(c => c.State === "running")
     .map(async (c) => {
       const container = docker.getContainer(c.Id)
       const stats = await container.stats({ stream: false })
       return { id: c.Id, stats }
     })
   ```
5. Transform each container into a clean response object:
   ```typescript
   {
     id: container.Id,
     name: container.Names[0]?.replace(/^\//, "") ?? "unknown",
     image: container.Image,
     state: container.State,      // "running", "exited", "paused", etc.
     status: container.Status,    // human-readable like "Up 3 hours"
     created: container.Created,  // unix timestamp
     ports: container.Ports.map(p => ({
       privatePort: p.PrivatePort,
       publicPort: p.PublicPort,
       type: p.Type
     })),
     stats: {
       cpuPercent: calculateCpuPercent(stats),
       memoryUsage: stats?.memory_stats?.usage ?? 0,
       memoryLimit: stats?.memory_stats?.limit ?? 0,
       memoryPercent: calculateMemPercent(stats)
     }
   }
   ```
6. CPU percentage calculation (this is the standard Docker formula):
   ```typescript
   function calculateCpuPercent(stats: any): number {
     if (!stats?.cpu_stats?.cpu_usage?.total_usage) return 0
     const cpuDelta = stats.cpu_stats.cpu_usage.total_usage - (stats.precpu_stats?.cpu_usage?.total_usage ?? 0)
     const systemDelta = stats.cpu_stats.system_cpu_usage - (stats.precpu_stats?.system_cpu_usage ?? 0)
     const cpuCount = stats.cpu_stats.online_cpus ?? stats.cpu_stats.cpu_usage?.percpu_usage?.length ?? 1
     if (systemDelta <= 0 || cpuDelta < 0) return 0
     return (cpuDelta / systemDelta) * cpuCount * 100
   }
   ```
7. Memory percentage:
   ```typescript
   function calculateMemPercent(stats: any): number {
     if (!stats?.memory_stats?.usage || !stats?.memory_stats?.limit) return 0
     return (stats.memory_stats.usage / stats.memory_stats.limit) * 100
   }
   ```
8. For stopped containers (State !== "running"), stats should be null/zeroed.
9. Return as JSON with `Cache-Control: no-store`.
10. Wrap in try/catch. Return 500 on unexpected errors.

IMPORTANT: `container.stats({ stream: false })` returns a Promise that resolves to the stats object directly. The `stream: false` flag is critical -- without it, stats returns a readable stream.

IMPORTANT: Stats collection can be slow with many containers. Set a timeout of 5 seconds on the overall operation. If stats take too long, return containers without stats rather than failing entirely.

**Route 2: POST /api/widgets/docker/[id] (container actions)**

Create `app/api/widgets/docker/[id]/route.ts` with a POST handler.

Request body:
- `action`: `"start"` | `"stop"` | `"restart"` (required)
- `socketPath` (optional): custom Docker socket path

Handler logic:
1. Extract `id` from dynamic route params (remember: params are Promises in Next.js 15+, must await).
2. Parse the request body and validate `action` is one of the three allowed values. Return 400 if invalid.
3. Create Dockerode instance and get the container: `docker.getContainer(id)`
4. Execute the action:
   ```typescript
   switch (action) {
     case "start": await container.start(); break
     case "stop": await container.stop(); break
     case "restart": await container.restart(); break
   }
   ```
5. Return `{ success: true, action, containerId: id }` with status 200.
6. If the container is not found, return 404.
7. If the action fails (e.g., trying to start an already running container), catch the Docker error and return 409 with the error message.
8. Wrap in try/catch. Return 500 on unexpected errors.

Use this pattern for the route context type (matching existing conventions):
```typescript
type RouteContext = { params: Promise<{ id: string }> }
```

function formatBytes(bytes: number, decimals = 1): string {
  if (bytes === 0) return "0 B"
  const k = 1024
  const sizes = ["B", "KB", "MB", "GB", "TB"]
  const i = Math.floor(Math.log(bytes) / Math.log(k))

}
