Create `app/api/auth/login/route.ts`:

POST handler:
- Accept `{ password: string, rememberMe?: boolean }`
- Check `isPasswordSet()` — if false, return 400 `{ error: "No password configured. Use /api/auth/setup" }`
- Get the password hash from settings: `db.select().from(settings).where(eq(settings.key, "passwordHash")).get()`
- Call `verifyPassword(password, hash)` — if false, return 401 `{ error: "Invalid password" }`
- Call `createSession(rememberMe ?? false)` to get a token
- Call `setSessionCookie(token, rememberMe ?? false)` to set the cookie — this must be awaited (async in Next.js 16)
- Return 200 `{ success: true }`

IMPORTANT: `setSessionCookie` uses `cookies()` which is async in Next.js 16, so this handler must await it.

Create `app/api/auth/setup/route.ts`:

POST handler — initial password creation (only works when no password is set):
- Accept `{ password: string, confirmPassword: string }`
- Check `isPasswordSet()` — if true, return 400 `{ error: "Password already configured" }`
- Validate password length >= 8 characters
- Validate password === confirmPassword
- Call `hashPassword(password)` and store in settings table: `db.insert(settings).values({ key: "passwordHash", value: hash }).run()`
- Create a session and set the cookie (auto-login after setup)
- Return 200 `{ success: true }`

Create `app/api/auth/logout/route.ts`:

POST handler:
- Get session from cookies via `getSessionFromCookies()`
- If session exists, call `deleteSession(session.token)`
- Call `clearSessionCookie()` — must be awaited
- Return 200 `{ success: true }`

Create `middleware.ts` at the project root (next to package.json):

```typescript
import { NextRequest, NextResponse } from "next/server";

export function middleware(request: NextRequest) {
  const sessionCookie = request.cookies.get("homelabarr_session");

  if (!sessionCookie?.value) {
    return NextResponse.redirect(new URL("/login", request.url));
  }

  return NextResponse.next();
}

export const config = {
  matcher: [
    /*
     * Match all paths except:
     * - /login (the login page itself)
     * - /api/auth/* (auth API endpoints)
     * - /_next/* (Next.js internals)
     * - /favicon.ico, /icons/*, etc. (static assets)
     */
    "/((?!login|api/auth|_next|favicon\\.ico|icons).*)",
  ],
};
```

NOTE: The middleware only checks cookie PRESENCE, not validity. Full session validation happens server-side in the API routes / server components. This is the standard Next.js middleware pattern — middleware runs on the edge and can't access SQLite. The cookie presence check is sufficient to redirect obvious unauthenticated requests; server-side validation catches expired/invalid sessions.
