  <action>
**Update `lib/services/types.ts`:**

Add two new auth types to the ServiceAuthType union:
- `"query-auth"` -- appends `?auth=TOKEN` as query param (Pi-hole pattern)
- `"header-basic-auth"` -- sets `Authorization: Basic base64(apiKey)` header (AdGuard pattern, where apiKey is pre-formatted as "user:pass")
- `"header-bearer"` -- sets `Authorization: Bearer TOKEN` header (Home Assistant pattern, will be used in Plan 03)
- `"header-pve-token"` -- sets `Authorization: PVEAPIToken=TOKEN` header (Proxmox pattern, will be used in Plan 02)

**Update `lib/services/service-client.ts`:**

Add cases to the switch statement in fetchService for each new auth type:
- `"query-auth"`: `url.searchParams.set("auth", apiKey)`
- `"header-basic-auth"`: `headers["Authorization"] = "Basic " + btoa(apiKey)` (apiKey is "user:pass")
- `"header-bearer"`: `headers["Authorization"] = "Bearer " + apiKey`
- `"header-pve-token"`: `headers["Authorization"] = "PVEAPIToken=" + apiKey`
  </action>

**Create `app/api/widgets/dns/types.ts`:**

Define normalized response types:
- `DnsClient` with fields: name (string), queryCount (number)
- `DnsStats` with fields: totalQueries (number), blockedQueries (number), blockPercentage (number), domainsBlocked (number), topClients (DnsClient[])
- `DnsResponse` with fields: stats (DnsStats), serviceType ("pihole" | "adguard"), protectionEnabled (boolean)

**Create `app/api/widgets/dns/route.ts`:**

GET handler following the Phase 8 pattern:
1. Extract widgetId from query params, validate, call getWidgetWithConfig
2. Read serviceType from config ("pihole" or "adguard")
3. Call getServiceConnection to get serviceUrl and apiKey

For Pi-hole (serviceType === "pihole"):
- Fetch stats: fetchService with endpoint "/admin/api.php", authType "query-auth", queryParams: { summaryRaw: "" }
  The Pi-hole API uses presence of the key, so pass `summaryRaw` as empty string query param alongside auth
- Fetch top clients: fetchService with endpoint "/admin/api.php", authType "query-auth", queryParams: { getQuerySources: "" }
  Note: Pi-hole puts auth in query params. The "query-auth" auth type adds `?auth=TOKEN`. Additional queryParams like `summaryRaw` get appended.
- Map Pi-hole response to DnsStats:
  totalQueries = data.dns_queries_today
  blockedQueries = data.ads_blocked_today
  blockPercentage = data.ads_percentage_today
  domainsBlocked = data.domains_being_blocked
  topClients = parse top_sources object (keys are "IP|hostname", values are query counts) into DnsClient[], take top 5

For AdGuard (serviceType === "adguard"):
- Fetch stats: fetchService with endpoint "/control/stats", authType "header-basic-auth"
  apiKey is "username:password" format stored as single secret
- Fetch status: fetchService with endpoint "/control/status", authType "header-basic-auth"
- Map AdGuard response to DnsStats:
  totalQueries = data.num_dns_queries
  blockedQueries = data.num_blocked_filtering
  blockPercentage = (blockedQueries / totalQueries) * 100 (compute it)
  domainsBlocked = 0 (AdGuard doesn't expose this in the stats endpoint)
  topClients = data.top_clients array, take top 5. Each entry is an object like { "192.168.1.1": 500 } — parse key as name, value as queryCount
  protectionEnabled from /control/status response

Return DnsResponse JSON with Cache-Control: no-store.
