Sonarr API v3 reference:
- Calendar: GET /api/v3/calendar?start={iso}&end={iso} -- returns array of episode objects
  - Episode: { id, seriesId, episodeNumber, seasonNumber, title, airDateUtc, series: { title, images }, hasFile }
- Queue: GET /api/v3/queue?page=1&pageSize=20&includeEpisode=true&includeSeries=true
  - Queue item: { id, title, status, trackedDownloadStatus, trackedDownloadState, sizeleft, size, timeleft, episode: {...}, series: { title } }
  - Status values: "downloading", "paused", "queued", "completed", "warning", "failed"

Radarr API v3 reference:
- Calendar: GET /api/v3/calendar?start={iso}&end={iso} -- returns array of movie objects
  - Movie: { id, title, year, digitalRelease, physicalRelease, inCinemas, hasFile, images }
- Queue: GET /api/v3/queue?page=1&pageSize=20&includeMovie=true
  - Queue item: { id, title, status, trackedDownloadStatus, trackedDownloadState, sizeleft, size, timeleft, movie: { title, year } }
  - Same status values as Sonarr

**Types file: `app/api/widgets/media-management/types.ts`**

Define normalized response types:

```typescript
export interface CalendarItem {
  id: number
  title: string            // episode title or movie title
  seriesOrMovieTitle: string // series name (Sonarr) or same as title (Radarr)
  episodeInfo?: string     // "S02E05" format for Sonarr, undefined for Radarr
  airDate: string          // ISO date string
  hasFile: boolean         // already downloaded?
  mediaType: "episode" | "movie"
}

export interface QueueItem {
  id: number
  title: string            // what is being downloaded
  seriesOrMovieTitle: string
  status: string           // "downloading" | "paused" | "queued" | "completed" | "warning" | "failed"
  progressPercent: number  // 0-100
  sizeTotal: number        // bytes
  sizeLeft: number         // bytes
  timeLeft: string         // "02:15:30" or similar from API, empty if unknown
  mediaType: "episode" | "movie"
}

export interface MediaManagementResponse {
  calendar: CalendarItem[]
  queue: QueueItem[]
  serviceType: "sonarr" | "radarr"
}
```

**API route: `app/api/widgets/media-management/route.ts`**

Create a GET handler that:

1. Reads query param: `widgetId` (required)
2. Loads widget config from DB using `getWidgetWithConfig(widgetId)` from helpers
3. Extracts: `serviceUrl`, `secretName`, `serviceType` ("sonarr" or "radarr")
4. Uses `getServiceConnection(config)` to resolve API key
5. If not configured, return 400

**Calendar fetch**:
- Build date range: start = today (ISO), end = 7 days from now (ISO)
- Endpoint: `/api/v3/calendar?start=${start}&end=${end}`
- Auth type: `"header-x-api-key"`

For Sonarr, map each episode:
- title: `episode.title`
- seriesOrMovieTitle: `episode.series?.title ?? "Unknown"`
- episodeInfo: `S${String(episode.seasonNumber).padStart(2,"0")}E${String(episode.episodeNumber).padStart(2,"0")}`
- airDate: `episode.airDateUtc`
- hasFile: `episode.hasFile`
- mediaType: `"episode"`

For Radarr, map each movie:
- title: `movie.title`
- seriesOrMovieTitle: `movie.title`
- episodeInfo: undefined
- airDate: `movie.digitalRelease ?? movie.physicalRelease ?? movie.inCinemas ?? ""`
- hasFile: `movie.hasFile`
- mediaType: `"movie"`

Sort calendar items by airDate ascending.

**Queue fetch**:
- Endpoint: `/api/v3/queue?page=1&pageSize=20` + Sonarr adds `&includeEpisode=true&includeSeries=true`, Radarr adds `&includeMovie=true`
- Response has `{ records: [...] }` wrapper

For Sonarr queue items:
- title: `item.episode?.title ?? item.title`
- seriesOrMovieTitle: `item.series?.title ?? "Unknown"`
- status: `item.trackedDownloadState ?? item.status ?? "queued"`
- progressPercent: calculate from size/sizeleft: `item.size > 0 ? ((item.size - item.sizeleft) / item.size) * 100 : 0`
- sizeTotal: `item.size`
- sizeLeft: `item.sizeleft`
- timeLeft: `item.timeleft ?? ""`
- mediaType: `"episode"`

For Radarr queue items:
- title: `item.movie?.title ?? item.title`
- seriesOrMovieTitle: `item.movie?.title ?? "Unknown"`
- Same status/progress/size mapping as Sonarr
- mediaType: `"movie"`

Return the combined MediaManagementResponse. If either sub-fetch fails, return empty array for that section. Cache-Control: no-store.

Create `components/widgets/media-management-widget.tsx` as a `"use client"` component.

Props: `{ widgetId: string, config: Record<string, unknown> | null }`

Config shape:
- `serviceUrl`: string
- `secretName`: string
- `serviceType`: "sonarr" | "radarr"

**Setup state** (when not configured):
Show a setup form with:
- Service type selector: dropdown with "Sonarr" and "Radarr" options
- Service URL input (placeholder: "http://192.168.1.100:8989" for Sonarr, "http://192.168.1.100:7878" for Radarr)
- Secret name input (placeholder: "sonarr-api-key")
- Save button that PATCHes config

**Configured state**:
- Header: library/collection icon (use LibraryIcon or Bookmark02Icon from hugeicons) + service type label ("Sonarr" / "Radarr") + gear icon
- Two-tab view: "Calendar" (default) and "Queue"

**Calendar view**:
- Group items by date (today, tomorrow, then by date)
- Each date group has a header: "Today", "Tomorrow", or formatted date (e.g., "Wed, Mar 5")
- Each item in a group:
  - Left: colored dot (green if hasFile, gray if not)
  - Middle: seriesOrMovieTitle (bold) + title and episodeInfo below in muted text
  - Right: air time (formatted from airDate, just time portion like "8:00 PM")
- Empty state: "No upcoming content in the next 7 days"

**Queue view**:
- Each queue item as a row:
  - Status indicator: colored dot (green=downloading, amber=queued/paused, red=failed, blue=completed)
  - Title (bold) + seriesOrMovieTitle (muted) stacked
  - Progress bar beneath the title area (thin, full-width within row)
  - Right side: percentage + size remaining (e.g., "45% - 2.3 GB left") + time left if available
- Empty state: "Download queue is empty"

**Polling**: every 60 seconds (queue changes less frequently than now-playing).

**Settings panel**: same fields as setup, pre-filled, with save button.

Visual style matches existing widget patterns. Progress bar for queue items uses:
- `bg-green-500` for downloading
- `bg-amber-500` for paused/queued
- `bg-red-500` for failed
- `bg-blue-500` for completed

Use the same `formatBytes` helper as other widgets for size display. Use `timeAgo` style display for times.

Format queue time-left: the API returns strings like "02:15:30" -- display as-is if present, or "Unknown" if empty.
