Install icon libraries (Simple Icons for brands, Lucide for general icons) and build the icon rendering infrastructure so app cards can display rich SVG icons.

Purpose: Replaces the current emoji-only icon system with a searchable library of 4000+ icons. This plan sets up the data layer and rendering; Plan 04 adds the picker UI.

Output: An AppIcon component that renders Simple Icons (brand logos), Lucide icons (general purpose), or emojis based on a prefixed string format, plus a searchable icon index for the picker.

The `icon` field in apps table is `text | null`. Currently stores emoji strings or null.

**Icon format convention (new):**
- `si:slug` -> Simple Icons brand icon (e.g., `si:plex`, `si:sonarr`, `si:jellyfin`)
- `lucide:name` -> Lucide icon (e.g., `lucide:home`, `lucide:server`, `lucide:globe`)
- No prefix -> treat as emoji (backward compatible)
- null/empty -> first-letter fallback

**Simple Icons API:**
- `import { siPlex } from 'simple-icons'` returns `{ title, slug, hex, path, svg }`
- `path` is the SVG path data, `hex` is the brand color
- ~3400 brand icons

**Lucide React API:**
- `import { DynamicIcon, iconNames } from 'lucide-react/dynamic'`
- `<DynamicIcon name="home" size={20} />` renders the icon
- `iconNames` is an array of all available icon name strings
- ~1500 general icons

Both libraries together provide ~4900 icons.

Install the two new dependencies:

```
npm install simple-icons lucide-react
```

Note: `lucide-react` may already be a transitive dependency of shadcn, but we need it as a direct dependency for the dynamic import and `iconNames` export.

Create `lib/icons/index.ts` -- the icon resolver module:

Export a function `parseIconRef(ref: string | null | undefined)` that returns:
- `{ type: "simple", slug: string }` if ref starts with `si:`
- `{ type: "lucide", name: string }` if ref starts with `lucide:`
- `{ type: "emoji", value: string }` if ref is non-empty and has no prefix
- `{ type: "none" }` if ref is null, undefined, or empty

Export a function `getSimpleIconData(slug: string): { title: string; path: string; hex: string } | null` that is SERVER-ONLY (used by API routes, not client components). This function:
- Converts the slug to PascalCase and prepends `si` to form the import key (e.g., `plex` -> `siPlex`)
- Imports from `simple-icons` and looks up the icon by that key
- Returns `{ title, path, hex }` or null if not found
- Caches the full import at module level so it only loads once

The `simple-icons` package exports each icon as `si[PascalCase]` from its main module. Each icon object has `{ title, slug, hex, path, svg }` properties. Only `path` and `hex` are needed for rendering.

Create `lib/icons/icon-data.ts` -- the search index for the picker (used by Plan 04):

This file exports two functions:
- `getSimpleIconEntries(): { slug: string; title: string; hex: string }[]` -- returns all Simple Icons metadata (no SVG paths, just search data). Import `* as simpleIcons` from `simple-icons` and map over the entries.
- `getLucideIconNames(): string[]` -- re-exports `iconNames` from `lucide-react/dynamic`

These are used by the icon picker to build the searchable list. They're relatively heavy imports, but only loaded when the picker dialog opens.
