<objective>
Build a searchable icon picker dialog and integrate it into the app form, replacing the plain text input with a rich icon selection experience.

Purpose: Users need to browse and search thousands of icons to find the right one for each app. This is the primary user-facing feature for the icon library.

Output: An IconPickerDialog component with tabbed browsing (Brands/General), real-time search, and a preview of the selected icon in the app form.
</objective>

  <name>Task 1: Build server-side icon search endpoint</name>
  <files>app/api/icons/search/route.ts</files>
  <action>
Create `app/api/icons/search/route.ts` -- a GET endpoint that returns icon search results.

Query params:
- `q` (required): search query string (min 1 char)
- `library` (optional): `simple` or `lucide` (default: both)
- `limit` (optional): max results (default: 60)

Logic:
1. Import `getSimpleIconEntries` from `@/lib/icons/icon-data` and `getLucideIconNames` from the same module
2. Cache the full icon lists at module level (they only need to load once)
3. Lowercase the query, then filter icons whose title/name includes the query string
4. For Simple Icons: return `{ ref: "si:${slug}", title, hex }` for each match
5. For Lucide: convert hyphenated names to title case for display (e.g., "arrow-right" -> "Arrow Right"), return `{ ref: "lucide:${name}", title: displayName }`
6. Combine results (Simple Icons first if both), limit to the specified max
7. Return as `{ icons: [...] }`

If `q` is empty or missing, return an empty array (don't return all 4000+ icons).

For initial browsing (when the picker opens with no search query), the picker will show a curated set of popular homelab icons. This is handled client-side in the picker component, not in this endpoint.
  </action>

- Results are capped at the limit
- Empty query returns empty results
  </verify>

- Results include ref string (si:slug or lucide:name), display title, and hex color (for Simple Icons)

**Layout:**
- Use a Dialog (from shadcn) at `sm:max-w-lg` width
- DialogHeader with title "Choose Icon"
- A search Input at the top (auto-focused)
- Tabs component with two tabs: "Brands" (Simple Icons) and "General" (Lucide)
- Below the tabs, a ScrollArea (max height ~320px) containing a grid of icon buttons

**Search behavior:**
- Debounce the search input (300ms) using a useRef timeout pattern (same as NotesWidget)
- On search query change, fetch GET `/api/icons/search?q=${query}&library=${activeTab === "Brands" ? "simple" : "lucide"}&limit=60`
- Display results in a grid (6 columns, gap-2)
- Each icon button is a small square showing the icon (using AppIcon component for consistent rendering) with the title below it as a tooltip

**Initial state (no search query):**
- Show a curated grid of popular homelab service icons. Hard-code a list of ~30 popular slugs:
