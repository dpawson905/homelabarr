<objective>
Add export/import UI to the settings page so users can download and upload dashboard configurations.

Purpose: Gives users a visual interface to back up, restore, and migrate their dashboards without touching API endpoints directly.

Output: A new "Configuration" card on the settings page with export button, file upload, mode selector, and result feedback.
</objective>

<context>
@.planning/PROJECT.md
@.planning/ROADMAP.md
@.planning/STATE.md

The settings page is at `app/settings/page.tsx`. It's a client component ("use client") that currently has two Card sections: Security (password change + logout) and Secrets Manager.

The page already imports and uses: Button, Input, Label, Dialog, AlertDialog, Card, Separator, Table, toast (sonner), HugeiconsIcon with various icons from @hugeicons/core-free-icons.

The export API (Plan 01) provides:
- GET /api/config/export -> JSON file download with Content-Disposition header
- POST /api/config/import -> accepts { config: object, mode: "merge" | "replace" }, returns { boardsImported, widgetsImported, appsImported, settingsImported, warnings }

The existing page pattern: each section is a self-contained function component (PasswordChangeForm, SecretsManager) composed inside the main SettingsPage export.
</context>

<task type="auto">
  <name>Task 2: Add import functionality with mode selection and confirmation</name>
  <files>app/settings/page.tsx</files>
  <action>
Extend the `ConfigExportImport` component with import functionality:

**File selection:**
- A hidden `<input type="file" accept=".json">` triggered by an "Import Configuration" button
- When a file is selected, read it with FileReader, parse the JSON, and store it in state
- Validate that the parsed JSON has a `version` field (show toast error if not valid)

**Mode selection and confirmation:**
- After a valid file is loaded, show an inline section (not a separate dialog) displaying:
  - The filename
  - A brief summary: "X boards, Y apps" (count from the parsed data)
  - Two buttons: "Merge" (adds to existing) and "Replace" (overwrites everything)
- The "Replace" button should be styled as destructive (variant="destructive") and wrapped in an AlertDialog confirmation: "This will delete all existing boards, widgets, and apps. Your password and secrets will be preserved. Continue?"
- "Merge" can proceed without extra confirmation

**Import execution:**
- POST to /api/config/import with `{ config: parsedData, mode: selectedMode }`
- Show loading state during import
- On success, show a toast with the summary: "Imported X boards, Y widgets, Z apps"
- If the result includes warnings, show them as additional toast messages (type: "warning" or just info)
- After successful import, call `router.refresh()` to update the sidebar and any cached data
- Clear the file selection state after import (success or failure)

**Layout in settings page:**
Add the new `ConfigExportImport` component as a third Card between the existing Security card and the Secrets card. Give it a CardHeader with title "Configuration" and description "Export or import your dashboard configuration."
  </action>
